# Internal Communications

## Purpose
This skill enables writing various internal company communications using standardized formats.

## Applicable Communication Types

This skill covers multiple internal communication formats:
- **3P updates (Progress, Plans, Problems)** - team progress updates
- **Company newsletters** - organization-wide announcements
- **FAQ responses** - answering common questions
- **Status reports** - project and initiative updates
- **Leadership updates** - executive communications
- **Project updates** - detailed project progress reports
- **Incident reports** - post-mortem and incident communications

## How to Use This Skill

### Step 1: Identify Communication Type
Determine which type of internal communication you need to create.

### Step 2: Follow Format Guidelines
Each communication type has specific formatting, tone, and content requirements:

#### 3P Updates (Progress, Plans, Problems)
Use for regular team updates, typically weekly or bi-weekly.

**Format:**
```
## Progress
- List completed items and achievements
- Include metrics and outcomes where applicable
- Highlight wins and milestones

## Plans
- Outline upcoming work and priorities
- Include timelines and deadlines
- Note any dependencies

## Problems
- Identify blockers and challenges
- Describe impact and urgency
- Request specific help or decisions needed
```

**Tone:** Direct, factual, action-oriented

#### Company Newsletter
Use for organization-wide announcements and updates.

**Format:**
```
## [Compelling Headline]

### Opening
Brief context or introduction

### Key Updates
- Update 1: [Title]
  Description and impact

- Update 2: [Title]
  Description and impact

### Looking Ahead
Preview of upcoming items

### Closing
Call to action or next steps
```

**Tone:** Engaging, inclusive, positive while authentic

#### FAQ Answers
Use for responding to common questions from teams.

**Format:**
```
Q: [Question as asked]

A: [Clear, concise answer]

   [Additional context if needed]

   [Related resources or links]
```

**Tone:** Helpful, clear, comprehensive

#### Status Reports
Use for project or initiative progress updates.

**Format:**
```
## Executive Summary
High-level status and key points

## Progress Since Last Update
Completed items and milestones

## Current Status
- Overall health: [On track / At risk / Behind]
- Key metrics
- Resource utilization

## Upcoming Milestones
Timeline and deliverables

## Risks and Issues
Challenges requiring attention

## Decisions Needed
Items requiring stakeholder input
```

**Tone:** Professional, transparent, data-driven

#### General Communication Guidelines

**Content Gathering:**
Before writing, collect:
- Key facts and data points
- Stakeholder perspectives
- Timeline information
- Action items and owners
- Relevant context and background

**Formatting Best Practices:**
- Use clear headings and structure
- Keep paragraphs short and scannable
- Use bullet points for lists
- Include relevant metrics
- Add visual breaks for readability
- Bold key information

**Tone Considerations:**
- Match the organization's communication culture
- Be authentic and transparent
- Balance positivity with realism
- Use inclusive language
- Be action-oriented
- Consider the audience

## When Format Is Unclear

If the communication request doesn't align with existing guidelines:
1. Ask clarifying questions about:
   - Target audience
   - Purpose and goals
   - Preferred format and length
   - Key messages to convey
   - Required sections or elements
2. Suggest the closest matching format
3. Adapt based on feedback

## Best Practices

### Be Clear and Concise
- Lead with most important information
- Use plain language
- Avoid jargon unless universal to audience
- One main idea per paragraph

### Provide Context
- Explain "why" not just "what"
- Connect to broader goals
- Acknowledge relevant background

### Make It Actionable
- Clear next steps
- Specific owners and deadlines
- Easy-to-find CTAs

### Maintain Consistency
- Follow organizational style guides
- Use consistent formatting
- Maintain regular cadence for recurring comms

### Proofread and Review
- Check facts and figures
- Verify names and titles
- Ensure tone is appropriate
- Confirm all links work
