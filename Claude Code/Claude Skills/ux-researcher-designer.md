# UX Researcher & Designer

Comprehensive toolkit for user-centered research and experience design.

## Core Capabilities

### 1. Data-Driven Persona Generation

#### What Are Personas?
Research-backed representations of target users that help teams:
- Understand user needs and behaviors
- Make design decisions
- Prioritize features
- Communicate user insights

#### Persona Development Process

**Step 1: Data Collection**
- User interviews
- Surveys and questionnaires
- Analytics data
- Support tickets
- Sales conversations

**Step 2: Pattern Analysis**
- Behavioral patterns
- Demographics
- Goals and motivations
- Pain points and frustrations
- Context of use

**Step 3: Persona Creation**
- Identify archetypes (3-5 primary personas)
- Document characteristics
- Create scenarios
- Extract design implications

**Step 4: Validation**
- Review with stakeholders
- Test with real users
- Refine based on feedback
- Assign confidence scores based on sample size

### 3. Usability Testing Frameworks

#### Test Planning

**Define Objectives**:
- What are you testing?
- What questions need answers?
- Success criteria?

**Select Method**:
- Moderated vs unmoderated
- Remote vs in-person
- Think-aloud protocol
- Task-based testing
- A/B testing

**Recruit Participants**:
- Target 5-8 users per round
- Match persona criteria
- Appropriate screener questions

#### Test Structure

**Introduction** (5 min):
- Welcome and consent
- Explain process
- Set expectations

**Pre-test Questions** (5 min):
- Background information
- Current behavior
- Expectations

**Tasks** (20-30 min):
- 3-5 realistic scenarios
- Observe and take notes
- Ask follow-up questions

**Post-test Questions** (10 min):
- Overall impressions
- Satisfaction ratings
- Suggestions

**Debrief** (5 min):
- Thank participant
- Clarify any questions

#### Metrics to Track

**Behavioral Metrics**:
- Task success rate
- Time on task
- Error rate
- Number of clicks/steps
- Help requests

**Attitudinal Metrics**:
- System Usability Scale (SUS)
- Net Promoter Score (NPS)
- Satisfaction ratings
- Perceived ease of use

### 4. Research Synthesis

#### Analysis Process

**Step 1: Organize Data**
- Transcribe recordings
- Compile notes
- Tag observations
- Categorize findings

**Step 2: Identify Patterns**
- Recurring themes
- Common pain points
- Unexpected insights
- Contradictions

**Step 3: Synthesize Insights**
- Group related findings
- Prioritize by impact
- Connect to business goals
- Generate recommendations

**Step 4: Create Deliverables**
- Research reports
- Insight presentations
- Design recommendations
- Action items

#### Insight Framework

```
Observation:
What we saw/heard

Pattern:
How often it occurred

Impact:
Why it matters

Recommendation:
What to do about it

Priority:
High/Medium/Low

Evidence:
Supporting data/quotes
```

### 5. Design Validation Methods

#### Validation Approaches

**Concept Testing**:
- Early-stage idea validation
- Multiple concepts comparison
- Quick feedback loops

**Prototype Testing**:
- Low-fidelity wireframes
- High-fidelity prototypes
- Interactive mockups

**A/B Testing**:
- Variant comparison
- Statistical significance
- Quantitative validation

**Beta Testing**:
- Real-world usage
- Extended timeframe
- Feature validation

#### Validation Checklist

- [ ] Clear success criteria defined
- [ ] Appropriate method selected
- [ ] Representative users recruited
- [ ] Test protocol documented
- [ ] Metrics tracked
- [ ] Results analyzed
- [ ] Insights documented
- [ ] Recommendations provided

## Best Practices

### Research Quality
- Recruit representative users
- Avoid leading questions
- Record sessions when possible
- Take detailed notes
- Validate findings
- Share insights widely

### Design Validation
- Test early and often
- Use appropriate fidelity
- Focus on critical flows
- Measure what matters
- Iterate based on feedback
- Track changes over time

### Stakeholder Management
- Involve early
- Communicate regularly
- Show, don't just tell
- Tie insights to business goals
- Make recommendations actionable
- Build research culture
